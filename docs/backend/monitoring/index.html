Monitoring

Backend поставляется с локальным monitoring-контуром на Prometheus и Grafana. Он нужен для быстрой проверки состояния сервисов, базовой диагностики нагрузки и наблюдения за очередями scoring/outbox.

Компоненты​

КомпонентЧто делает
apiотдает /metrics на HTTP-порту API
flagcheckerотдает /metrics на health-порту flagchecker
prometheusсобирает метрики приложения и инфраструктурных exporter-ов
grafanaпоказывает dashboard на основе Prometheus datasource
postgres-exporterэкспортирует метрики PostgreSQL
redis-exporterэкспортирует метрики Redis
rabbitmq-exporterэкспортирует метрики очередей RabbitMQ

Запуск​

docker compose up -d --build

Основные адреса и targets:

СервисURL
Grafanahttp://127.0.0.1:3000
Prometheushttp://127.0.0.1:9090
API metricshttp://127.0.0.1:8080/metrics
FlagChecker metricshttp://flagchecker:9091/metrics внутри Docker-сети

По умолчанию Prometheus и Grafana публикуются только на 127.0.0.1. Перед внешней публикацией Grafana нужно заменить логин и пароль администратора через env-переменные.

Что смотреть в Grafana​

В репозитории есть provisioned dashboard Lost Last Overview. Сейчас в нем заведены такие панели:

API P95 Latencyp95 latency по всем HTTP-запросам

RPS by RouteRPS в разрезе route

Полезные PromQL-запросы​

Общий RPS API:

sum(rate(lost_last_http_requests_total[5m]))

p95 latency API:

histogram_quantile(0.95, sum by (le) (rate(lost_last_http_request_duration_seconds_bucket[5m])))

Доля 5xx:

sum(rate(lost_last_http_requests_total{status=~"5.."}[5m])) / clamp_min(sum(rate(lost_last_http_requests_total[5m])), 1)

RPS по route:

sum by (route) (rate(lost_last_http_requests_total[5m]))

Outbox backlog:

lost_last_outbox_events_pending + lost_last_outbox_events_retry

RabbitMQ scoring queue:

rabbitmq_queue_messages{queue="scoring.events"}

Hit ratio кеша лидерборда:

sum(rate(lost_last_leaderboard_cache_requests_total{result="hit"}[5m])) / clamp_min(sum(rate(lost_last_leaderboard_cache_requests_total{result=~"hit|miss"}[5m])), 1)

Проверка на сервере​

После обновления кода на сервере:

docker compose pull
docker compose up -d --build

Быстрая проверка:

docker compose ps
curl -fsS http://127.0.0.1:8080/metrics | head
curl -fsS http://127.0.0.1:9090/-/ready
curl -fsS http://127.0.0.1:3000/api/health

Примечания​

node-exporter и cadvisor не включены в основной compose, потому что они зависят от хоста и делают локальный запуск тяжелее. Их стоит добавлять отдельно на Linux-сервере, если нужны host/container metrics.
