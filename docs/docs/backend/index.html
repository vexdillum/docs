Vexd

Vexd - backend-платформа для проведения соревнований формата Jeopardy CTF.

Проект объединяет классические функции CTF-платформы с управляющим контуром для динамических заданий и отдельным сервисом проверки флагов.

Основной акцент проекта - устойчивость во время пиковых отправок флагов, разделение пользовательского API и flag checking, кэширование read-heavy данных и подготовка к управлению task-инстансами через kCTF/Kubernetes.

Возможности​

Для участников платформа предоставляет полный базовый цикл CTF:

  • регистрация и вход в аккаунт;
  • создание команды и вступление в существующую команду;
  • просмотр профиля, состава команды и прогресса;
  • просмотр каталога заданий по категориям;
  • отправка флагов и получение результата проверки;
  • просмотр таблицы лидеров и последних решенных заданий.

Для организаторов реализован административный контур:

  • создание, обновление и удаление заданий;
  • управление видимостью, стоимостью и параметрами scoring;
  • старт, завершение и freeze/unfreeze соревнования;
  • бан, разбан и жесткое удаление пользователей;
  • просмотр команд, пользователей, solve log и audit log;
  • управление активными инстансами динамических заданий.

Для устойчивости под нагрузкой используются отдельный flagchecker, очередь событий, Redis-кэширование leaderboard/dashboard и фоновые пересчеты scoring.

Быстрый запуск​

Требования:

  • Docker;
  • Docker Compose;
  • Go 1.25 для локальных тестов без контейнеров.
Copy-Item .env.example .env
docker compose up -d --build

Проверка API:

Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8080/ping

Ожидаемый ответ:

{"message":"pong"}

По умолчанию наружу публикуются только пользовательские HTTP-точки и мониторинг на localhost. PostgreSQL, Redis, RabbitMQ и flagchecker доступны внутри Docker-сети.

API-документация​

Swagger UI лежит в репозитории и открывается как статический файл:

Invoke-Item docs\swagger.html

OpenAPI JSON:

docs/openapi.json

Перегенерация API-документации:

go run ./cmd/docs

Качество и CI/CD​

CI/CD в GitLab запускает форматирование, vet, golangci-lint, тесты, coverage и Docker build.

Основные команды:

gofmt -w .
go vet ./...
golangci-lint run ./... --timeout 5m

Форк kCTF проверяется отдельно из своего Go-модуля:

Push-Location kctf\kctf-operator
golangci-lint run ./... --timeout 5m
Pop-Location

Coverage считается по основным внутренним пакетам, без main, generated code, router wiring и генератора документации:

go test -covermode=atomic -coverprofile coverage.out `
  ./internal/config `
  ./internal/flagcheckerapi `
  ./internal/infra/... `
  ./internal/scoring `
  ./internal/service/... `
  ./internal/transport/grpc/flagchecker `
  ./internal/transport/httpx `
  ./internal/transport/mw `
  ./internal/util/slug

go tool cover -func coverage.out
