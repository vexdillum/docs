# Работа с фронтендом

Frontend находится в папке `lost-last-frontend/`. Это Angular-приложение
Lost Last: каталог заданий, страницы команд и пользователей, рейтинг, профили,
авторизация и административные интерфейсы.

## Требования

- Node.js 24 или новее.
- npm 10 или новее.

Рекомендуемая версия Node.js зафиксирована в `.nvmrc`, `.node-version`,
`package.json` и GitLab CI.

## Установка

```bash
nvm use
npm ci
```

Если `nvm` не установлен, поставьте Node.js 24+ вручную и затем выполните `npm ci`.

## Локальный запуск

```bash
npm start
```

По умолчанию Angular запускает приложение на `http://localhost:4200`. Базовый
URL backend API задается в `src/shared/config/api.config.ts`; для локальной
разработки с backend используется `proxy.conf.json`.

## Сборка

Production-сборка:

```bash
npm run build
```

Результат появляется в `dist/CTFast/browser`.

Development-сборка:

```bash
npm run build:development
```

## Проверки

Перед merge request нужно выполнить:

```bash
npm run lint
npm run test:coverage
npm run build
```

Назначение команд:

| Команда | Что делает |
| --- | --- |
| `npm run lint` | Проверяет TypeScript и Angular-шаблоны |
| `npm test` | Запускает unit-тесты |
| `npm run test:coverage` | Запускает unit-тесты с покрытием |
| `npm run activity:check` | Проверяет Git-активность для технической защиты |
| `npm run docs` | Генерирует Compodoc в `documentation/` |

## Структура фронтенда

```text
src/app/              Bootstrap Angular-приложения, маршруты и providers
src/pages/            Страницы верхнего уровня
src/widgets/          Переиспользуемые UI-виджеты
src/entities/         Общие TypeScript-интерфейсы
src/utils/            API-клиенты, guards и interceptors
src/shared/config/    Runtime-конфигурация
src/assets/i18n/      Словари локализации
public/               Статические файлы для сборки
```

## Фронтенд-документация

```bash
npm run docs
```

Команда собирает Compodoc HTML в `documentation/`. В общем портале также
публикуется справочник [Фронтенд-документация](../frontend/autogenerated):
компоненты, сервисы, интерфейсы и маршруты Angular-приложения.
