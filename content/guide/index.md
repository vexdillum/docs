# Руководство по работе с Lost Last

Lost Last состоит из frontend-приложения, backend-платформы и отдельного сайта документации.
Этот раздел описывает основной рабочий цикл: как подготовить окружение, поднять проект,
проверить его и собрать артефакты.

## Структура репозиториев

| Часть | Папка | Назначение |
| --- | --- | --- |
| Фронтенд | `lost-last-frontend/` | Angular-приложение для участников и администраторов |
| Бэкенд | `lost-last/` | Go backend, API, flagchecker, инфраструктурные сервисы |
| Документация | `docs-site/` | Docusaurus-сайт с руководством, contribution, архитектурой и API |

## Быстрый порядок работы

1. Поднять бэкенд через Docker Compose.
2. Установить зависимости фронтенда и запустить Angular-приложение.
3. Проверить качество фронтенда и бэкенда перед merge request.
4. Обновить автодокументацию и собрать `docs-site/docs/` перед публикацией документации.

## Основные команды

| Задача | Где выполнять | Команда |
| --- | --- | --- |
| Запустить бэкенд | `lost-last/` | `docker compose up -d --build` |
| Проверить backend API | `lost-last/` | `curl http://127.0.0.1:8080/ping` |
| Установить frontend-зависимости | `lost-last-frontend/` | `npm ci` |
| Запустить frontend | `lost-last-frontend/` | `npm start` |
| Собрать frontend | `lost-last-frontend/` | `npm run build` |
| Собрать документацию | `docs-site/` | `npm run build` |
