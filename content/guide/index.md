# Руководство по работе с Lost Last

Lost Last состоит из трех рабочих частей: бэкенд, фронтенд и TG bot.
Этот раздел описывает основной рабочий цикл: как подготовить окружение, поднять проект,
проверить его и собрать артефакты.

## Структура репозиториев

| Часть | Папка | Назначение |
| --- | --- | --- |
| Бэкенд | `lost-last/` | Go-бэкенд, API, flagchecker, инфраструктурные сервисы |
| Фронтенд | `lost-last-frontend/` | Angular-приложение для участников и администраторов |
| TG bot | `lost-last-tg-bot/` | Заглушка Telegram-бота поддержки для выполнения ТЗ |

## Быстрый порядок работы

1. Поднять бэкенд через Docker Compose.
2. Установить зависимости фронтенда и запустить Angular-приложение.
3. При необходимости запустить TG bot локально или через Docker.
4. Проверить качество всех затронутых частей перед merge request.

## Основные команды

| Задача | Где выполнять | Команда |
| --- | --- | --- |
| Запустить бэкенд | `lost-last/` | `docker compose up -d --build` |
| Проверить API бэкенда | `lost-last/` | `curl http://127.0.0.1:8080/ping` |
| Установить зависимости фронтенда | `lost-last-frontend/` | `npm ci` |
| Запустить фронтенд | `lost-last-frontend/` | `npm start` |
| Собрать фронтенд | `lost-last-frontend/` | `npm run build` |
| Запустить TG bot локально | `lost-last-tg-bot/supportbot/` | `python -m app.main` |
| Запустить TG bot через Docker | `lost-last-tg-bot/supportbot/` | `docker compose up -d --build` |
