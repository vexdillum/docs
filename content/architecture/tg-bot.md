# Архитектура TG bot

TG bot — отдельная заглушка сервиса поддержки. Он не интегрирован напрямую с
backend Lost Last, но показывает наличие дополнительного сервиса вокруг основной
CTF-платформы.

## Назначение

Бот принимает обращения участников в личном чате и пересылает их в группу
поддержки. Операторы отвечают из группы, а бот отправляет ответы обратно
участнику.

## Стек

| Часть | Технология |
| --- | --- |
| Язык | Python 3.12+ |
| Telegram framework | aiogram 3 |
| Хранилище | SQLite |
| Запуск | Локально или через Docker |

## Основные модули

```text
supportbot/app/config.py      Конфигурация окружения
supportbot/app/db.py          Работа с SQLite
supportbot/app/keyboards.py   Telegram-кнопки
supportbot/app/main.py        Точка входа
supportbot/app/models.py      Модели данных
supportbot/tests/             Pytest-тесты
```

## Поток обращения

1. Участник пишет боту и создает обращение.
2. Бот сохраняет тикет и сообщение в SQLite.
3. Бот публикует обращение в группе поддержки.
4. Оператор отвечает reply-сообщением в группе.
5. Бот пересылает ответ участнику.
6. Обращение закрывается кнопкой или командой `/close`.

## Ограничения

- Нет связи с пользователями и командами из backend.
- Нет общей авторизации с платформой.
- SQLite выбран как простой локальный storage.
- Сервис нужен как демонстрационная часть проекта под требования ТЗ.
