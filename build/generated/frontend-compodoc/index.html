En

Lost Last Frontend

Frontend for a CTF platform. The application shows challenges, teams, users, team pages, user profiles, rating tables and authentication pages.

Stack

  • Angular 21
  • TypeScript 5.9
  • Carbon Design System
  • Tailwind CSS
  • Transloco for localization
  • ECharts for charts
  • Vitest and Angular test runner
  • ESLint and angular-eslint

Requirements

Use Node.js 24 or newer and npm 10 or newer. The recommended runtime is fixed in .nvmrc, .node-version, package.json and GitLab CI.

Example :
nvm use
npm ci

If nvm is not installed, install Node.js 24+ manually and then run npm ci.

Development

Start the local frontend:

Example :
npm start

By default Angular starts the app at http://localhost:4200.

The frontend API base URL is configured in src/shared/config/api.config.ts. For local backend proxying, use proxy.conf.json if the backend is started on a separate port.

Scripts

Example :
npm run lint
npm test
npm run test:coverage
npm run docs
npm run build
npm run build:development
  • lint checks TypeScript and Angular templates.
  • test runs unit tests.
  • test:coverage runs unit tests with coverage.
  • docs generates Compodoc HTML documentation in documentation/ from Angular component metadata, including inputs and outputs.
  • build creates a production build in dist/CTFast/browser.
  • build:development creates a development build.

Project Structure

Example :
src/app/              Angular app bootstrap, routes and providers
src/pages/            Route-level pages
src/widgets/          Reusable UI widgets
src/entities/         Shared TypeScript interfaces
src/utils/            API clients, guards and interceptors
src/shared/config/    Runtime configuration
src/assets/i18n/      Localization dictionaries
src/test-*.ts         Shared test setup
public/               Static assets copied to the build

CI/CD

GitLab CI is configured in .gitlab-ci.yml.

Pipeline stages:

  1. quality: runs ESLint.
  2. test: runs ng test --coverage --watch=false and publishes coverage/ artifacts.
  3. build: creates the production frontend bundle.
  4. docs: generates Compodoc HTML documentation in documentation/.

Pipeline artifacts:

  • coverage/
  • dist/CTFast/browser/
  • documentation/

Pipeline Scheme

Example :
GitLab commit -> GitLab CI -> npm ci -> lint -> тесты с покрытием -> production build -> Compodoc docs

The pipeline is reproducible because all dependencies are installed from package-lock.json, Node.js is fixed in CI, and both the static build output and generated documentation are produced directly by CI.

Git Workflow

The team uses GitLab Flow with dev as the integration branch and main as the release branch. Detailed branch, commit and merge request rules are described in CONTRIBUTING.md.
